    Linux ‐ OpenSSL

    • Course related: Cryptography and Cryptanalysis
    • This project demonstrates practical applications of cryptographic techniques using OpenSSL on Linux systems. Our objectives included exploring symmetric encryption, key generation, message encryption/decryption, and digital signatures. Through a series of tasks, we utilized OpenSSL to manage both symmetric (AES-128-CBC, AES-256-CTR, DES) and asymmetric (RSA 2048-bit) encryption.

      • We began by creating and sharing public/private key pairs, emphasizing the importance of securing private keys with encryption. The project involved:
      • Symmetric Encryption: Testing AES-128-CBC and AES-256-CTR algorithms for encrypting and decrypting messages, including performance benchmarking.
      • Asymmetric Encryption: Generating RSA key pairs, encrypting messages with public keys, and decrypting with private keys. We exchanged encrypted messages to verify the process.
        • Digital Signatures: Signing and verifying messages to ensure authenticity and integrity using OpenSSL's RSA utility.

    Linux ‐ Secure Communication Using OpenSSL and Certificate Authority

    • Course related: Cryptography and Cryptanalysis
    • This project focuses on establishing a secure communication channel using OpenSSL and the Certificate Authority (CA) method. The primary objective was to ensure confidentiality, integrity, and authentication in the message exchange between two entities, Ron (Alice) and Lucas (Bob).

      • The project involved the following key steps:
      • Key Generation: Each participant generated their own RSA public and private key pairs. Additionally, keys were generated for the CA to facilitate certificate creation and verification.
      • Certificate Creation: Both participants created and signed their certificates using the CA, adding an extra layer of trust and authentication.
      • Message Encryption and Decryption: Messages were encrypted with the sender's private key and decrypted using the recipient's public key, ensuring secure transmission. Symmetric keys were also utilized for encrypting larger messages, with the symmetric keys themselves being encrypted using RSA keys.
      • Verification and Integrity Checks: Certificates were exchanged and verified to authenticate the identities of both parties, preventing unauthorized access and ensuring message integrity.
